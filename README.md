# AI Dev Agent (`ai_dev`)

A lightweight, autonomous C++ terminal agent loop that integrates the Gemini API directly into your Linux terminal. It acts as an intelligent subshell executor capable of inspecting your workspace, exploring files, and performing complex multi-file codebase operations natively.

---

## 🚀 Features

* **Autonomous Exploration:** Runs non-destructive system exploration commands (`ls`, `cat`, `grep`, `find`) automatically to gather project context.
* **Intelligent Key Rotation:** Automatically cycles through a comma-separated list of multiple Gemini API keys if a `429 RESOURCE_EXHAUSTED` or rate-limiting error occurs.
* **One-Shot Edit Policy:** Safely bundles all context-driven file operations into a single clean batch write execution instead of risky incremental loops.
* **Shell Agnostic:** Fully compatible with interactive shells like `Fish`, while safely delegating execution to standard, universal `Bash` subshells.
* **Ansi-Colorized Terminal Output:** Provides immediate feedback with explicit, stylized indicators for API metrics, steps, and key shifts.

---

## 🛠️ Requirements & Dependencies

Before building, ensure you have the following system packages installed on your Linux machine:

* **Compiler:** G++ supporting C++11 or higher.
* **JSON Processor:** `jq` installed on your system path (used for parsing API responses).
* **Networking:** `curl` command-line utility with SSL support.

To install dependencies on Debian/Ubuntu/Chromebook (Crostini):

```
sudo apt-get update && sudo apt-get install build-essential jq curl
```

## 📦 Installation & Setup
### 1. Build from Source
Compile the ai_engine.cpp file into an executable file binary:
```
g++ -std=c++11 -O3 ai_engine.cpp -o ai_dev
```
Move the binary into your local system execution path (e.g., /usr/local/bin or ~/bin) to access it globally.

### 2. Configuration & API Credentials
The program searches for configuration parameters inside your environment variables first, then falls back to a global config file inside your home folder.

Create a file named `~/.ai_config.json` using your favorite text editor:;/
```
{
  "GEMINI_API_KEY": "ghp_your_first_key_here,ghp_your_second_key_here",
  "GEMINI_MODEL": "gemini-2.5-flash"
}
```
## 💻 Usage
Run the executable and append your development request or prompt as command-line arguments:
```
ai_dev "Review the Python files in this folder and fix any syntax issues or broken configurations"
```

### ⚠️ Security Notice
* Local Executions: This application automatically executes code snippets generated by a generative model directly within a local subshell. Always review workspace contexts carefully when operating on sensitive environments.

* **Exclusions:** Keep your setup files, private tokens, or system configurations backed up or wrapped safely away from active directory scopes during heavy processing batches.
